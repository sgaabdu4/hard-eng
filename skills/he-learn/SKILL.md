# Hard Eng Learn

## Contract

- Input = fresh PLAN + open learning candidate OR explicit `$he learn` evidence.
- Invocation = explicit `$he learn` OR `$he`/stage route with verified trigger; exposed metadata ≠ unsolicited classification.
- Output = proven narrow prevention + candidate receipt OR exact non-candidate/blocker.
- Lifecycle = unchanged; this skill is an overlay, never a stage or route target.
- Load [workflow.md](references/workflow.md) before classification or promotion.

## Ownership

- Candidate + promotion + final consolidation = this skill.
- Recurrence/root-cause proof = `$repeated-failure-learning`.
- Implementation/proof = current lifecycle stage owner.
- State/checkpoint = `$he`; mutation in ship → `$he-build` final round.

## Invariants

- Trigger enum = `recurrence|user-correction|systemic-critical-gap|false-gate|repeated-manual-waste` + verified evidence.
- One-off implementation defect = normal slice issue; learning candidate = forbidden.
- Final audit first discovers a required plan state/boundary/scenario = `false-gate`; repeated material final-audit class = `systemic-critical-gap|repeated-manual-waste`.
- Memory = optional recall; PLAN + repository owner + deterministic proof = SSOT.
- Cross-scope prevention = source candidate + exact destination proof link; bounded known repair → destination direct; unresolved destination decision → `$he`.
- Source pause = sole source PLAN writer lease; destination repair never mutates source PLAN; source read/proof allowed, checkpoint/reapproval/candidate apply forbidden until handback.
- Source lifecycle = paused, never nested; resume only after destination proof or explicit deferral.

## Complete

- Root failure class + evidence + owner + required proof = explicit.
- Narrowest authorized prevention = implemented + regression-proven.
- Candidate = atomically resolved; open candidate blocks `shipped`.
