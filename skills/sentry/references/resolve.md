# Resolve

1. Prove deployed release/commit + affected IDs + expected observation window.
2. Gather fresh events after stated traffic/time window → recurrence or observation limit explicit.
3. Deployed proof + exact approval → `issue resolve <issue> --in <release|@commit|@next>`.
4. Fresh issue view → remote status + resolution target verified per ID.

- Missing deployed/observation/approval proof → stop with exact next owner/action.
- Complete = every scoped ID verified fixed, deferred, or blocked; no local-only proof presented as production proof.
