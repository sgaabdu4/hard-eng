# Triage Workflow

## Disclaimer

Every comment or issue posted during triage starts with:

```markdown
> *This was generated by AI during triage.*
```

## References

- `AGENT-BRIEF.md`: durable agent brief format
- `OUT-OF-SCOPE.md`: rejected enhancement knowledge base

## Roles

Category roles:

- `bug`
- `enhancement`

State roles:

- `needs-triage`
- `needs-info`
- `ready-for-agent`
- `ready-for-human`
- `wontfix`

If roles conflict, ask the maintainer before changing them.

## Show Attention Queue

Query the tracker and present:

- unlabeled items
- `needs-triage` items
- `needs-info` items with reporter activity since the last triage note

When external PRs are in scope, include them and label each row `[PR]` or `[issue]`.

## Triage An Item

Gather context:

- issue or PR body, comments, labels, author, and dates
- PR diff when triaging a PR
- prior triage notes
- relevant code and domain docs
- `.out-of-scope/*.md`

Check redundancy by searching for existing implementation by domain concept, not just prompt wording.

Recommend category and state with evidence, then wait for direction.

Verify:

- reproduce bugs from reporter steps when possible
- confirm PR diffs do what they claim
- report confirmed, failed, or insufficient detail

Shape when needed:

- use `grill-me` for one-question-at-a-time alignment
- use `domain-modeling` to sharpen terms and ADRs

Apply outcome:

- `ready-for-agent`: post an agent brief
- `ready-for-human`: post an agent brief structure and explain why human work is needed
- `needs-info`: post triage notes with established facts and exact questions
- `wontfix`: close with reason
- `needs-triage`: apply the role and optionally comment

For already-implemented requests, close as `wontfix` and point to the implementation. Do not write `.out-of-scope/` for already-built behavior.

For rejected enhancements, write `.out-of-scope/` and link it from the close comment.

## Override

If the maintainer explicitly says to move an item to a state, trust them. Confirm the exact action, apply it, and ask whether they want an agent brief when moving straight to `ready-for-agent`.

## Resume

When prior triage notes exist, read them first, check whether the reporter answered outstanding questions, and avoid re-asking resolved questions.
