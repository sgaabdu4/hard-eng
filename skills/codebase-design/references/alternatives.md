# Interface Alternatives

1. Fix one chosen boundary + invariant constraints + dependency categories + caller needs.
2. Produce 2–3 materially different contracts: minimal surface; common-caller optimized; flexibility only when current variation proves it.
3. Show each contract + caller example + hidden behavior + dependency strategy + error/order rules + deletable concepts.
4. Compare depth + locality + coupling + migration/blast radius + testability; reject cosmetic variants.
5. Recommend one option or hybrid + why + revisit trigger; material choice → user decision through `$he`.

- Complete = selected contract, or exact undecided trade-off + next decision owner.
