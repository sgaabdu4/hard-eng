# Repository Stage

Decision inventory = repository root + canonical remote/project identity + branch + base/HEAD revision + feature identity + root `PRODUCT.md`/`DESIGN.md` status + relevant related repositories.

1. Resolve requested feature → candidate repository/worktree.
2. Verify Git identity natively → capture branch + immutable revisions.
3. Run `$deterministic-checks` repository-context branch → capture valid/invalid evidence.
4. Missing/invalid pair → apply Stage Route product context + `$atomic-ui` DESIGN route: research code/docs/history/assets → ask unresolved intent → approve → write → validate.
5. Search workspace/docs/config → include or rule out coupled repositories; each Git repository owns its own root pair.
6. Reconcile mismatch/inaccessibility → one accepted identity + valid approved context pair or blocker.

Complete = every identity field proven + root pair valid/current/approved; repository stage cannot skip.
