# Research Workflow

## Evidence Order

Use local repo facts first for codebase questions: `rg`, codebase tools,
checked-in docs, diffs, config, and tests.

Use web/search/extraction for current sources, URL discovery, official docs,
APIs, standards, changelogs, papers, vendor pages, and source repos. Prefer
primary sources over summaries.

## Rules

- Define the research question and the decision it should unblock
- Use a background agent only for substantial reading that can proceed in
  parallel with main work
- Gather primary sources first; use secondary sources only for context gaps
- Separate `Verified`, `Inferred`, and `Unknown`
- Cite URLs or local file paths for every material claim
- Check dates and primary sources before trusting old docs, blogs, or AI
  summaries as current
- Save notes only when they will be reused by the project
- Match the repo convention for saved notes; if none exists, choose a sensible
  Markdown path and report it

## Output

Answer the decision question first. List sources with dates or versions when
available. Call out unknowns and the next proof needed.
