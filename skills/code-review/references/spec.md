# Spec Axis

1. Read user request + PR description before judging intent.
2. Resolve source: user path/content → supplied issue/PR → commit-linked issue/PR via repository tracker contract → matching `docs/`, `specs/`, `.scratch/` plan/spec.
3. No behavioral source → obtain user confirmation or skip Spec + state unknown.
4. Compare diff/public behavior/contracts/routes/schemas/fixtures/tests against quoted requirements.
5. Report missing/partial requirement + scope creep + wrong implementation with distinct spec evidence.
